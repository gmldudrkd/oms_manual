# User List

From the left-hand menu **User → User List**, you can review user accounts and their permission status, and approve or reject permission requests.

---

## Searching for Users

| Search Field | Description |
|-----------|------|
| **Search By** | Search criteria — Email / Name / ID |
| Search term | Enter a value matching the criteria (for Email, select a domain and enter the ID) |
| **Approve Status Filter** | Approval status — AWAITING / APPROVAL / REJECTED |

### List Columns

| Column | Description |
|------|------|
| ID | User ID |
| Requested Date | Date and time the sign-up request was made |
| **Email** | Click to go to the user detail page |
| **Approval Status** | Sign-up approval status (AWAITING = orange, APPROVAL = green) |
| **Permissions** | Summary of held permissions (expand to view details) |

---

## Viewing Permission Status

The **Permissions** field summarizes a user's permissions as a count per role. Click it to expand the roles by brand and corporation, shown as chips.

| Role Chip | Color | Meaning |
|---------|------|------|
| **Admin** | Blue | Administrative permission |
| **Manager** | Green | General operational handling |
| **Request** | Orange | Awaiting approval |
| **None** | Gray | No permission |

:::tip
To check "which brands and corporations this person can handle," expand the Permissions field. It shows at a glance which role applies to which corporation for each brand.
:::
