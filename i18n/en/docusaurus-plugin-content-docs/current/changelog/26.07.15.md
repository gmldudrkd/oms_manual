# 26.07.15

> Feature release for GM KR integration and operational improvements.

---

## 1. Order Details – Improved Per-Product Information Display

Previously, bundles were grouped as a single line without their components, and paid items such as lenses and packages were not managed individually — making it hard to check per-product prices and claim-processing history.

- **Individual rows** – Products in Ordered Product Info are now displayed as separate rows per product.
- **Bundle components shown** – Single products included in a bundle are each shown as their own row.
- **Paid-item prices shown** – Items sold for a fee, such as packages and lenses, are displayed as individual rows with their per-product price.
- **Reshipped quantity check** – For individual Claim–Reshipment processing, you can check per-product how many of which product were reshipped.

:::note
Products are shown separately, but NUF bundle separation is not yet supported.
:::

## 2. Return Details – Pickup Status / Collection Method Display

Return intake cases (Force Refund / Pickup Requested / Not Requested / BORIS) vary, but the existing return details screen could not distinguish how a return was received.

- **Pickup field added** – The return details show whether pickup is `Not Requested` / `Requested`.
- **FORCE REFUND info shown** – `FORCE REFUND` is added at the top of the return details.
- **Excel reflected** – Force-refund cases can be checked in the Excel export.

| Intake Case | Pickup | ReturnMethod |
| --- | --- | --- |
| Force Refund | Not Requested | `FORCE REFUND` |
| Return + Pickup Not Requested | Not Requested | `PARCEL` |
| Return + Pickup Requested | Requested | `PARCEL` |
| BORIS | Not Requested | `IN STORE` |

## 3. Exchange – Package-Only Exchange Support

GM has many defective-package cases, so exchange requests are frequent. Korea only accepts package exchanges, but previously the package could not be exchanged separately from the main product.

- **Package-only exchange** – On an exchange request, only the package can be separated and exchanged.

![exchagne package Info](/img/exchagne_package.png)

## 4. Exchange – Automatic Shipment on Stock Allocation

When there is no exchange shipment stock, the item is held in `Inspected` status, requiring Request Shipment to be handled manually — repeating the sales → operations → CX steps every time.

- **Automatic allocation & shipment** – Shipments in Inspected status are treated as unallocated shipments and are automatically allocated and shipped during the once-daily closing-stock receipt and distribution.
- **Manual handling removed** – The manual Request Shipment step, previously required whenever stock was unavailable, is no longer needed.

:::note
Exchange shipment stock allocation is based on the stock distributed to the channel.
When unallocated at shipment) Allocated first from the quantity received as closing stock, before it is distributed to channels.
When unallocated at exchange shipment) Allocated from the stock quantity distributed to each channel from the closing stock (not shipped if there is no distributed stock).
:::

## 5. Shipment Awaiting Status Added

Farfetch, an external GM KR channel, issues its own tracking numbers, so a step to confirm the tracking number is needed after stock allocation (Collected). This step is handled as the `Shipment Awaiting` status.

- **Guide shown** – The `Shipment Awaiting` status and its description are shown in the Status Guide.

## 6. Return – Lost Return SAP Integration

SAP did not distinguish lost items separately, so losses were also processed as revenue (−) / stock (+), and operators had to manually pull the order list and handle it separately. A LOST warehouse is now set up in SAP to process lost returns there.

- **Lost info transmitted** – When the return reason is sent as `Lost`, the lost-related information is passed to SAP and processed in the LOST warehouse.

## 7. Stock – Instant Channel Transmission of Stock Quantity Added

This feature lets you instantly transmit the current Available quantity to a channel, used when you need to temporarily pause sales or reflect stock after a pre-order ends.
It can be applied to only one channel at a time, based on the Available quantity.

![send available qty](/img/send_available_qty.png)

## 8. Order – Shipping Fee Shown in Order Export

As brands with shipping fees (ATS overseas delivery, NUF, etc.) were added to OMS, the shipping-fee item needed to be checked when extracting order data.

- **Shipping Fee item added** – On order Excel export, a Shipping Fee item is shown next to Tax.
- **Shown on all rows** – The shipping fee per order is displayed on all product rows.
