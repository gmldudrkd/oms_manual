# 26.03.25 Release Notes

> Additional features released for GM integration.

### 1. Overview Improvements

1. Order / Claim tabs separated
    - Improved visibility after Store Pickup and Reshipment were added
2. Store Pickup and Reshipment sections added

### 2. Order List - Type, Tag, and Received Method Fields Added

1. Received Method, Type, and Tag can now be checked in the order list.

    - Received Method, Type, and Tag information

        | Order Information | ReceiveMethod | Type | Tag |
        | --- | --- | --- | --- |
        | Delivery | `DELIVERY` |  |  |
        | Delivery + Gift | `DELIVERY` | `GIFT` |  |
        | Pickup | `STORE_PICKUP` |  |  |
        | Pre-order + Pickup | `STORE_PICKUP` |  | `PRE-ORDER` |
        | RX | `DELIVERY` | `RX` |  |
        | Pre-order | `DELIVERY` |  | `PRE-ORDER` |
        | LENS_ONLY | `DELIVERY` | `LENS_ONLY` |  |
        | Pre-order + Pickup + Gift | `STORE_PICKUP` | `GIFT` | `PRE-ORDER` |

2. Filters added for the new fields

### 3. Register Claim Feature Added

1. Pickup Option can be selected for return/exchange pickup (received tracking number is required)
    - `Claims that have already been received can be processed`
    - `If the requested SKU differs from the received SKU, cancel and re-register the claim, then proceed without pickup`

2. Reshipment feature added
    - `Shipment-only processing is available without pickup for defects, losses, etc.`
    - Similar to manual shipment

### 4. Reshipment Menu Added

1. Order > Reshipment List menu added -> `Displays information registered through Register Claim`

### 5. Shipment Loss Handling Feature Added

1. Lost button is shown in Shipped status -> `Force Refund` or `Reshipment` can be processed

2. Process `Force Refund` or `Reshipment`

### 6. Manual Grading After Receipt Added

1. Manual grading after return/exchange receipt is provided only for corporations using local logistics
    - `Return/exchange claims in received status` -> click Refund for returns or Inspected for exchanges to open the modal

### 7. Exchange Info Feature Added

1. Exchange shipment before receipt added -> `Advance shipment before product receipt is available`

2. Exchange shipment status can be checked

3. `Inspected` status added to exchange claims

    **Inspected**

    - Because exchanges are finalized only after shipment is completed, not immediately after receipt, grading completion is now kept as the `Inspected` status.
    - When exchange shipment is completed, the status changes to Exchanged.

    **Changes**

    - AS-IS: After exchange product receipt and grading completion -> `Exchanged`
    - TO-BE: After exchange product receipt and grading completion -> `Inspected`
        - Exchange claim status after Inspected, depending on channel stock and exchange shipment progress

            | Channel stock available \ Exchange shipment in progress | O | X |
            | --- | --- | --- |
            | O | Exchange shipment delivered -> Exchanged, exchange shipment in progress -> Shipment Requested | Shipment Requested |
            | X | Exchange shipment delivered -> Exchanged, exchange shipment in progress -> Shipment Requested | Inspected |

    **Reference**

    - https://www.figma.com/board/igmzSzMCjqT8tBKD72ziFF/Diagrams?node-id=1912-2687&t=8VlfOiF9hzpC7Jk-4

### 8. User Account Search Feature Added

1. For Admin users
    1. User > User List added
2. For individual users, My Info can be used to request additional brand/entity permissions when the user already has an account

    1. `Request My Additional Permissions` feature added in My Info

        My Info details

### 9. Shipment Number Copy Feature Added

### 10. Channel Stock Is Used for Reshipment Caused by Exchange Shipment, Claim Shipment, Etc.

**Changes**

- AS-IS: Stock was not used for reshipment caused by exchange, defects, loss, etc., when the shipment was not based on an order. Shipment always proceeded.
- TO-BE: Channel stock is used when reshipment occurs due to exchange, defects, loss, etc., even if the shipment is not based on an order.

**Caution**

- If there is no channel stock, reshipment due to exchange, defects, loss, etc. is not available.
- Move stock to fill channel stock, then process manual shipment.
