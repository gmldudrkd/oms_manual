# Glossary

This is a summary of terms that appear frequently in the manual and on screen. Since the on-screen labels are in English, the terms are sorted by their English names.

## Organization · Structure

| Term | Description |
|------|------|
| **Brand** | A sales brand (GENTLE MONSTER, TAMBURINS, ATIISSU, NUFLAAT, etc.) |
| **Corp** | The national corporation for sales and settlement (KR, US, JP, CA, TW, SG, AU, etc.) |
| **Channel** | The actual sales outlet. The Official store or a Marketplace (KAKAO, SSG, FARFETCH, etc.) |
| **Role** | The role: Admin / Manager / Request / None |

## Product · Stock

| Term | Description |
|------|------|
| **SKU** | The code for a product's minimum sellable unit (including color and size distinctions) |
| **SAP Code** | The product code managed in SAP (ERP) |
| **UPC Code** | The barcode (product identification) code |
| **Bundle** | A set product created by combining several Single items |
| **Online Stock** | Total available stock at the brand/corporation level (before distribution to channels) |
| **Channel Stock** | Stock distributed to a specific channel |
| **Safety Stock** | The minimum quantity set aside to prevent stockouts |
| **Distribution** | Dividing online stock and allocating it across channels |
| **Pre-order** | The pre-order quantity sold before stock arrives |

## Order · Shipment

| Term | Description |
|------|------|
| **Order** | A customer order |
| **Order No** | The OMS order number |
| **Origin/Purchase No** | The original order number from the sales channel |
| **Shipment** | A shipment (the packing/shipping unit). One order can be split into multiple shipments |
| **Store Pickup** | An in-store pickup order |
| **Receive Method** | The receiving method: Delivery or Store Pickup |
| **Order Type** | The order type: NORMAL / GIFT / RX |
| **Tracking No** | The courier tracking number |
| **WMS** | The warehouse management system (performs picking, packing, and shipping) |

## Claims (Return / Exchange / Cancellation)

| Term | Description |
|------|------|
| **Claim** | A collective term for cancellation, return, exchange, and reshipment requests |
| **Return** | A return (item collection + refund) |
| **Exchange** | An exchange (collecting the original item + sending a new one) |
| **Reshipment** | A reshipment (re-sending a failed or lost shipment) |
| **Pickup** | Collecting an item for a return or exchange |
| **Inspection / Grading** | Inspecting collected items and assigning a grade (A/B/C) |
| **Force Refund** | Processing an immediate refund without inspection |
| **Fault** | Liability: CUSTOMER (customer's fault) / OPERATION (operations' fault) |

## System

| Term | Description |
|------|------|
| **SAP (ERP)** | The settlement and finance system |
| **PIM** | The source system for product information |
| **Adapter** | The relay system that connects the OMS to external channels, warehouses, and the ERP |
| **Export** | The function for downloading data such as orders as an encrypted Excel file |
