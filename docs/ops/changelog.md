# Changelog

최근 시스템 업데이트로 변경되거나 추가된 기능입니다.

## 검수 완료 프로세스 통합 (OMS-2026)
- **변경 전**: 반품과 교환의 검수 완료를 별도의 로직으로 처리
- **변경 후**: 동일한 메서드로 통합하여 일관된 검수 처리 보장
- **영향**: 운영 관점에서 차이 없음, 시스템 안정성 향상

## 이벤트 필드 설정 추가 (OMS-2025)
- **내용**: Shipment, Exchange 이벤트에 필드 설정 코드 추가
- **영향**: 이벤트 추적 정확도 향상

## 주문 태그 기능 추가 (OMS-2045)
- **내용**: 주문에 `PREORDER`(사전주문) 태그를 부여할 수 있음
- **영향**: 사전주문 건을 별도로 필터링하여 관리 가능

## 수령 방법 구분 강화 (OMS-2017)
- **내용**: 주문 메시지에 수령 방법(배송/매장픽업)과 태그 정보 추가
- **영향**: 외부 시스템과의 연동 시 수령 방법 정보 정확도 향상

## 매장 픽업 상태 필터 추가 (OMS-2022)
- **내용**: 주문 목록 조회 시 매장 픽업 상태별 필터링 가능
- **영향**: 매장 픽업 주문의 진행 현황을 효율적으로 모니터링

## 채널 가용재고 조회 API (OMS-2002)
- **내용**: 특정 채널의 상품별 가용 재고를 한 번에 조회
- **영향**: 재고 확인이 더 빠르고 편리해짐

## 반송장 번호 검색 필터 (OMS-2020)
- **내용**: 반품/교환 목록 조회 시 반송장 번호로 검색 가능
- **영향**: 특정 반송장의 반품/교환 건을 빠르게 찾을 수 있음

## 재출고 관리 기능 확장 (OMS-1997, OMS-1998)
- **내용**: 교환/재출고에 대한 출고 취소, 분실, 거절 기능 추가
- **영향**: 출고 이후 발생하는 예외 상황을 체계적으로 관리 가능

## 매장 픽업 기능 (OMS-1913 ~ OMS-1917)
- **내용**: 매장 픽업 엔티티 생성, 이벤트 처리, 주문 이력 통합
- **영향**: 온라인 주문 → 매장 수령 시나리오 완전 지원

## 자가물류 법인 검수 지원 (OMS-1901)
- **내용**: WMS를 사용하지 않는 법인(CA, AU, TW, SG)에서 수동 검수 가능
- **영향**: 모든 법인에서 반품/교환 검수 완료 처리 가능

## 출고 분실 처리 (OMS-1911)
- **내용**: 배송 중(SHIPPED) 상태의 출고에 대해 분실 처리 기능 추가
- **영향**: 배송 분실 시 즉시 후속 처리(재출고 등) 가능

## 주문 이력 확장 (OMS-1996)
- **내용**: 주문 이력 조회 시 매장 픽업, 재출고 이력 정보 포함
- **영향**: 주문의 전체 라이프사이클을 한 눈에 파악 가능

## 주문 응답 필드 확장 (OMS-2000, OMS-2001)
- **내용**: 주문 목록/상세 조회 응답에 재출고 수량, 환불 결제, 매장 픽업 정보 추가
- **영향**: 더 상세한 주문 정보를 한 화면에서 확인 가능

## 주문 Export 유형 추가 (OMS-1999)
- **내용**: STORE_PICKUP, RESHIPMENT 엑셀 다운로드 유형 추가
- **영향**: 매장 픽업 및 재출고 데이터를 별도로 다운로드 가능

## 주문 대시보드 확장 (OMS-1995)
- **내용**: 주문 대시보드에 매장 픽업 집계 정보 추가
- **영향**: 매장 픽업 주문의 전체 현황을 대시보드에서 파악 가능
