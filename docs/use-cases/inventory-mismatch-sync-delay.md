# 재고 불일치/동기화 지연 (Inventory Mismatch)

> **상황**: 판매 채널과 OMS의 재고 수량이 다르거나, 주문이 OMS에 늦게 들어옵니다.

OMS는 외부 시스템(판매 채널, 창고, ERP)과 데이터를 주고받기 때문에, 동기화 타이밍 차이로 일시적 불일치가 발생할 수 있습니다.

## 왜 발생하나

| 현상 | 원인 |
|------|------|
| 주문이 OMS에 늦게 표시됨 | 채널 주문 수집이 **주기적으로** 일어남 (실시간 아님) |
| 채널엔 재고 있는데 OMS는 0 | 재고 분배 배치가 아직 반영 안 됨 |
| OMS 재고가 채널보다 많음 | 출고 확정이 일부 누락/지연 |
| 송장이 늦게 표시됨 | 창고(WMS)→OMS 상태 전달 지연 |

## 운영자 확인 순서

1. **헤더 Brand & Corp 확인** — 다른 법인을 보고 있어 비어 보이는 경우가 가장 흔합니다.
2. **잠시 대기 후 Refresh** — 주문 수집·상태 전달은 주기적이므로 시간이 지나면 반영됩니다.
3. **[재고 변동 이력](../stock/history) 확인** — 언제·왜 수량이 바뀌었는지 추적합니다.
4. **분배 설정 확인** — [채널 분배 설정](../stock/distribution-setting)의 비율이 의도대로인지 확인합니다.
5. **지속되면 시스템 담당자에게 에스컬레이션** — 일정 시간이 지나도 해소되지 않으면 연동(채널/창고/ERP) 문제일 수 있으므로 백엔드/인프라 담당자에게 전달합니다.

:::tip 예상 반영 시간 (참고)
- 주문 수집: 채널에 따라 다소 시간 소요(실시간 아님)
- 출고 상태/송장: 보통 수 분 내 반영
- 재고 분배: 일배치 기준으로 반영

위 시간이 크게 초과되면 단순 지연이 아닐 수 있으니 담당자에게 확인을 요청하세요.
:::
