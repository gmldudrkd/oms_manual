# 프로모션 등록·수정

프로모션 목록에서 신규 등록하거나, 기존 프로모션을 열어 수정합니다. 프로모션은 **발동 조건(Trigger)**과 **혜택(Reward)**을 설정하는 구조입니다.

---

## 신규 등록

목록 상단에서 신규 작성을 시작하면 **New(새로 작성)** 또는 **Copy(기존 복사)** 중 선택할 수 있습니다. Copy를 선택하면 기존 프로모션을 검색해 그 설정을 그대로 가져옵니다.

### 1. 기본 정보 (General)

| 항목 | 설명 | 필수 |
|------|------|:----:|
| **Title** | 프로모션 제목 (최대 50자) | ✅ |
| **Promotion Type** | GWP / Packaging Benefit | ✅ |
| **Brand / Corp** | 적용 브랜드·법인 | ✅ |
| **Start / End Date** | 시작·종료 일시 | ✅ |
| **Reason** | 등록 사유 (최대 50자) | ✅ |

### 2. 발동 조건 (Trigger Detail)

어떤 조건일 때 혜택을 줄지 정의합니다.

| 항목 | 설명 |
|------|------|
| **Trigger Type** | 일정 금액 이상 구매 / 특정 상품·라벨 구매 / 특정 상품 일정 금액 이상 / 모든 상품 구매 |
| **Amount / Currency** | 금액 조건이 있는 경우 기준 금액과 통화 |
| **Trigger Channel** | 프로모션을 적용할 채널(복수 선택) |
| **Trigger Product** | 특정 상품 조건일 때 대상 상품 지정 (모달에서 OR/AND 조건 구성) |
| **Exception Product** | 금액·전체 상품 조건일 때 제외할 상품 |

### 3. 혜택 (Reward Detail)

| 항목 | 설명 |
|------|------|
| **Reward Type** | Order Level(주문 단위) / Product Level(상품 단위) |
| **Reward Product** | 제공할 사은품/혜택 상품과 수량, 재고 사용 수량(Dedicated/Remained/Alert) 설정 |

---

## 저장과 검토

| 버튼 | 동작 |
|------|------|
| **Save Draft** | 작성 중 상태(Draft)로 임시 저장 |
| **Submit Review** | 모든 필수 항목 입력 후 검토 제출(확정 절차) |
| **Cancel** | 작성 취소 (변경분이 있으면 확인 창 표시) |

**Submit Review**를 누르면 입력 내용을 요약한 검토 창이 뜹니다. 제목·기간·발동 조건·혜택 상품을 최종 확인하고 **"Confirm"**으로 확정합니다.

---

## 수정 시 제약

이미 적용 중이거나 예정된 프로모션은 일부 항목만 수정할 수 있습니다.

- **Draft** 상태: 대부분 항목 수정 가능
- **Active / Upcoming / Expired** 상태: **End Date(종료일)**와 **혜택 수량** 등 제한된 항목만 수정 가능

---

## 프로모션 삭제

프로모션 상세 화면에서 **Draft 또는 Upcoming** 상태일 때만 삭제할 수 있습니다.

1. 상세에서 **"Delete"** 버튼을 클릭합니다.
2. 확인 창에 **`delete`**를 직접 입력해야 삭제 버튼이 활성화됩니다.

:::warning
삭제는 **영구적**이며 되돌릴 수 없습니다. 관련 설정과 데이터가 모두 제거됩니다.
:::
