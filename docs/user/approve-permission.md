# 권한 승인·반려 (관리자)

관리자(Admin)는 다른 사용자가 요청한 권한을 검토해 승인하거나 반려합니다.

:::warning 책임이 따르는 작업
권한 승인은 **승인한 관리자의 책임으로 기록**됩니다. 요청자가 해당 데이터에 접근해도 되는 사람인지 반드시 확인하세요.
:::

---

## 처리 방법

1. [User List](./user-list)에서 처리할 사용자의 **Email**을 클릭해 상세로 들어갑니다.
2. **Permission Requests(권한 요청)** 표에서 각 요청의 내용을 확인합니다.
   - Requested Date, Brand, Corp, Role, Requested Reason, Status
3. 상태가 **AWAITING**인 요청에 대해 **"Approve"**(승인) 또는 **"Reject"**(반려)를 선택합니다.

### 승인 (Approve)

- 확인 창에서 *"...This action will be recorded as your responsibility..."* 안내를 확인하고 **"Approve"**로 확정합니다.
- 승인되면 요청자는 즉시 해당 브랜드·법인 데이터에 접근할 수 있습니다.

### 반려 (Reject)

- 확인 창에서 *"...This will permanently delete the data."* 안내를 확인하고 **"Reject"**로 확정합니다.
- 반려된 요청 데이터는 삭제됩니다.

---

## 가입 승인

신규 가입자의 **Approval Status**가 **AWAITING**이면 아직 가입 승인 전입니다. 사용자 상세에서 가입 자체를 승인해야 로그인·권한 사용이 가능합니다.
