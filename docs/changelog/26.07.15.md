## 1. 주문 상세 – 제품별 정보 표시 개선

기존에는 번들이 구성품 없이 하나로 묶여 있고, 유상 판매되는 렌즈·패키지가 개별로 관리되지 않아 제품별 가격과 클레임 처리 내역을 확인하기 어려웠습니다.

- **개별 row 표시** – Ordered Product Info의 제품이 제품 단위 row로 분리 표시됩니다.
- **번들 구성품 노출** – 번들에 묶인 싱글 제품이 각각 별도 row로 표시됩니다.
- **유상 제품 가격 노출** – 패키지·렌즈처럼 유상으로 판매되는 제품이 개별 row로 표시되고, 제품별 가격이 함께 노출됩니다.
- **Reshipped 수량 확인** – 개별 Claim–Reshipment 처리 시 어떤 제품이 몇 개 재출고되었는지 제품별 수량을 확인할 수 있습니다.

:::note
제품 별 분리되어 보여지지만 NUF 번들 분리 처리는 아직 지원하지 않습니다.
:::

## 2. 반품 상세 – 픽업 여부 / 수거방식 표시

반품 접수 케이스(강제환불 / 픽업 요청 / 미요청 / BORIS)가 다양한데, 기존 반품 상세 화면에서는 어떤 방식으로 접수된 건인지 구분할 수 없었습니다.

- **Pickup 필드 추가** – 반품 상세에 픽업 여부가 `Not Requested` / `Requested`로 노출됩니다.
- **FORCE REFUND 정보 노출** – 반품 상세 상단에 `FORCE REFUND`(강제환불)가 추가됩니다.
- **엑셀 반영** – 엑셀 export 시 강제환불 건에 대한 내용을 확인할 수 있습니다.

| 접수 케이스 | Pickup | ReturnMethod |
| --- | --- | --- |
| 강제환불 | Not Requested | `FORCE REFUND` |
| Return + 픽업 미요청 | Not Requested | `PARCEL` |
| Return + 픽업 요청 | Requested | `PARCEL` |
| BORIS | Not Requested | `IN STORE` |

## 3. 교환 – 패키지 단독 교환 지원

GM은 패키지 불량 건이 많아 교환 요청 빈도가 높습니다. 한국은 패키지만 교환을 받아주고 있으나, 기존에는 본품과 분리해 패키지만 교환 처리할 수 없었습니다.

- **패키지 단독 교환** – 교환 요청 시 패키지만 분리하여 교환할 수 있도록 처리됩니다.

![exchagne package Info](/img/exchagne_package.png)

## 4. 교환 – 재고할당 시 자동 출고

교환 출고 재고가 없으면 `Inspected` 상태로 홀딩되는데, 이때 Request Shipment를 수기로 처리해야 해서 sales → 운영 → CX로 이어지는 처리 단계가 매번 반복됐습니다.

- **자동 할당·출고** – Inspected 상태의 출고 건은 미할당 출고로 간주되어, 하루 한 번 마감재고 수신·분배 시 자동으로 할당 후 출고됩니다.
- **수기 처리 제거** – 재고 없을 때마다 하던 Request Shipment 수기 처리가 필요 없어집니다.

## 5. Shipment Awaiting 상태 추가

GM KR 외부채널인 파페치는 송장을 자체 발행하기 때문에, 재고 할당(Collected) 이후 송장을 확인하는 단계가 필요합니다. 이 단계가 `Shipment Awaiting` 상태로 처리됩니다.

- **가이드 노출** – Status Guide에 `Shipment Awaiting` 상태와 설명이 노출됩니다.

## 6. 반품 – 분실 반품 SAP 연동

SAP은 분실을 별도로 구분하지 않아 분실도 매출(−)·재고(+)로 처리됐고, 운영자가 주문 리스트를 수기로 받아 별도 작업해야 했습니다. SAP에 LOST 창고를 두어 분실 반품을 이 창고에서 처리합니다.

- **분실 정보 전달** – 반품 사유를 `Lost`로 전송하면 SAP에 분실 관련 정보가 함께 전달되어 LOST 창고에서 처리됩니다.

## 7. 재고 – 프리오더 세팅 시 재고 분배 처리

매장픽업 B2B 출고용 재고가 온라인 창고에 미리 들어오면서, "온라인 창고에 재고 없음"이라는 프리오더 전제조건이 깨져 기능을 정상적으로 사용하지 못하는 문제가 있었습니다.

- **채널 분배비율 자동 0%** – 프리오더 세팅 시 해당 기간 동안 채널재고 분배비율이 자동으로 0%로 설정됩니다.
- **Shipped 재고 유지** – 프리오더 기간에는 마감재고가 수신되어도 Shipped 재고가 초기화되지 않습니다. (프리오더 출고재고는 내부적으로 별도 관리되며 UI상 Shipped 재고에 합산되어 노출)

## 8. 주문 – 주문정보 추출 시 Shipping Fee 노출

ATS 해외배송·NUF 등 배송비가 발생하는 브랜드가 OMS에 생기면서, 주문 데이터 추출 시 배송비 항목 확인이 필요해졌습니다.

- **Shipping Fee 항목 추가** – 주문 엑셀 추출 시 Tax 옆에 Shipping Fee 항목이 노출됩니다.
- **전체 row 노출** – 주문당 배송비가 전체 제품 row에 모두 표시됩니다.
