# 상태 전이 규칙의 배경

## 왜 특정 전이만 허용되는가?

### 주문 상태 전이

**PENDING → COLLECTED (확정)**
> 주문이 들어온 후 시스템이 주문 정보를 검증하고 수령인 정보를 확정하는 단계입니다. 이 시점에서 재고 할당이 시작됩니다.

**PENDING → DELETED (삭제)**
> 아직 아무 처리도 시작되지 않은 주문만 삭제할 수 있습니다. 재고 할당이나 출고가 시작된 후에는 삭제가 아닌 **취소(CANCELED)** 처리만 가능합니다. 삭제는 주문이 잘못 수신된 경우에 사용합니다.

**CANCELED → COMPLETED (취소 → 완료)**
> 부분 취소된 주문의 경우, 취소되지 않은 나머지 상품의 배송이 완료되면 전체 주문을 완료 처리합니다. 이 전이가 없으면 부분 취소된 주문은 영원히 완료될 수 없습니다.

### 출고 상태 전이

**PICKING_REQUESTED → PACKED (피킹 → 포장, PICKED 건너뜀)**
> 일부 물류 센터에서는 피킹과 포장을 한 번에 처리합니다. 단계를 건너뛸 수 있도록 유연하게 설계되었습니다.

**SHIPPED → LOST (배송 중 → 분실)**
> 배송 중에만 분실이 발생할 수 있습니다. 아직 배송 전이거나 이미 도착한 건은 분실 처리 대상이 아닙니다.

**PICKING_REJECTED에서만 취소/재출고 가능**
> 피킹이 거절된 상태에서만 출고 취소 요청이나 재출고 요청이 가능합니다. 피킹이 진행 중이거나 이미 배송된 건은 다른 방식으로 처리해야 합니다.

### 반품 상태 전이

**PENDING에서만 수령인 변경 가능**
> 수거 요청이 배송사에 전달된 후에는 주소를 변경하면 배송사와의 정보 불일치가 발생합니다. 따라서 수거 요청 전에만 주소 변경이 허용됩니다.

**PICKUP_REQUESTED/PICKUP_ONGOING/RECEIVED에서 검수 완료 가능**
> 검수는 상품이 물리적으로 도착해야 하지만, 자가물류 법인에서는 수거 과정을 직접 관리하므로 수거 요청 단계부터 검수 완료가 가능합니다.

### 교환 상태 전이

**교환은 반품보다 단계가 하나 더 많다 (INSPECTED)**
> 교환은 반품(수거) + 출고(배송) 두 가지 프로세스를 결합한 것입니다. 따라서 INSPECTED(검수 완료) → SHIPMENT_REQUESTED(출고 요청) 단계가 추가됩니다.

**출고 수령인 변경은 PENDING ~ PICKUP_ONGOING까지만**
> 교환 상품의 배송 주소는 출고가 시작되기 전까지만 변경 가능합니다. 이미 검수가 완료되어 출고 단계에 진입하면 물류 프로세스와 충돌할 수 있습니다.

## 재고 관련 규칙

**왜 비관적 락을 사용하는가?**
> 동일 상품에 대해 여러 주문이 동시에 들어올 때, 재고를 정확하게 차감하기 위해서입니다. 낙관적 락을 사용하면 재고가 마이너스가 되는 과매도 상황이 발생할 수 있습니다.

**왜 채널별로 재고를 분리하는가?**
> 각 판매 채널에서 독립적으로 재고를 관리하기 위해서입니다. 한 채널에서 재고가 소진되어도 다른 채널의 판매에 영향을 주지 않습니다. 필요 시 채널 간 재고 이전(Transfer)으로 유연하게 대응할 수 있습니다.

**왜 안전재고를 설정하는가?**
> 재고 동기화 지연이나 예상치 못한 대량 주문 상황에서 품절을 방지하기 위한 버퍼입니다. 안전재고만큼은 판매에서 제외되어 최소한의 재고가 항상 보장됩니다.
